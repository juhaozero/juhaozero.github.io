Spring入门——AOP(面向切面，切什么面？

SpringAOP

了解SpringAOP之前，我们先要了解一些关于代理的概念

什么是代理

代理(Proxy)是一种设计模式,提供了对目标对象另外的访问方式;即通过代理对象访问目标对象.这样做的好处是:可以在目标对象实现的基础上,增强额外的功能操作,即扩展目标对象的功能

简单点说 就是在不修改代码的情况下，对原有的一些功能进行增强。举个例子，当我们想吃饭时，以前我们会买菜，做饭，而现在我们只需要在点餐App上下单即可，外卖小哥就会将饭菜打包到我们这里来，我们省去了买菜，做饭这些时间，转而由饭店为我们代劳（差不多就是这个意思）。

Spring AOP 简介

如果说 SpringIOC 是 Spring 的核心，
那么SpringAOP即面向切面编程就是 Spring 最为重要的功能之一

什么是SpringAOP

所谓面向切面编程，就是我希望在目标对象（有无接口都可以）上扩展原有代码的功能，但可以在不修改源码的基础上来完成！，这听起来是不是有点耳熟，这不就是代理嘛！
SpringAOP是基于代理进行的。那什么是面向切面编程呢？？

百度百科上是这样介绍的

可以通过预编译方式和运行期动态代理实现在不修改源代码的情况下给程序动态统一添加功能的一种技术。AOP实际是GoF设计模式的延续，设计模式孜孜不倦追求的是调用者和被调用者之间的解耦,提高代码的灵活性和可扩展性，AOP可以说也是这种目标的一种实现。
在Spring中提供了面向切面编程的丰富支持，允许通过分离应用的业务逻辑与系统级服务（例如审计（auditing）和事务（transaction）管理）进行内聚性的开发。应用对象只实现它们应该做的——完成业务逻辑——仅此而已。它们并不负责（甚至是意识）其它的系统级关注点，例如日志或事务支持。

简单点说AOP实现了 ==通用的非核心业务逻辑，从核心业务逻辑中进行抽离==

举个简单例子：如果我们要做日志的管理，那我们需要在每一行的业务代码上添加日志管理的代码，但这造成了代码之间的耦合，但如果有一天我们不需要日志管理这个业务，我们又需要往业务代码上注释日志管理的代码，这样显然是不够方便的，我们就可以利用AOP进行编程.

SpringAOP的概念

  1. 连接点（jointPoint） 被代理对象的所有方法，即被代理的类下的方法
  2. 切入点（Poincut）被增强的方法就是切入点，即需要在哪个方法下添加其他业务逻辑
  3. 通知（Advice） 增强的代码，即非核心业务逻辑
  4. 切面（Apsect） 切面 = 通知+ 切入点
  5. 目标对象（Target）被代理对象
  6. 代理对象 (Proxy)

Aop编程通知类型

前置通知 (aop:before)
在执行目标对象方法之前执行

后置通知 (aop:after-returning)
在执行目标对象方法之后执行 (异常不执行)

异常通知(aop:after-throwing)
在执行目标对象方法里面发生异常时候执行

最终通知（aop:after）
在执行完目标对象方法后始终执行的方法就是最终通知(异常也执行)

环绕通知 (aop:around)
自定义如何执行目标方法

切入点表达式

在上面的案例中我们使用了切入点表达式
什么是切入点表达式呢?
切入点表达式：（用于声明要在目标对象的哪些方法进行切入通知！！）

execution( 修饰符 返回值类型 包路径.类名称.方法名称( 参数列表类型 ) )

  1. execution(): 表达式主体。
  2. 第一个*号：表示返回类型， *号表示所有的类型。
  3. 包名：表示需要拦截的包名，后面的两个句点表示当前包和当前包的所有子包，com.shelock.service.impl.AccountServiceImpl下的包、子孙包下所有类的方法。
  4. 第二个号：表示类名，号表示所有的类。
  5. (..):最后这个星号表示方法名，号表示所有的方法，后面括弧里面表示方法的参数，两个句点表示任何参数。
